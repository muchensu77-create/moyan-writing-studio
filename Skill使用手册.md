## 系统概述

Skill 系统是墨言工作室的内部运营工具集，用于标准化客户需求采集、订单管理和售后跟踪。

**三层架构：**

| 层级 | 用途 | 包含的 Skill |
|------|------|-------------|
| 客户接入 | 与客户沟通时采集需求信息 | Essay、Dissertation、Research、Application、Coding、Polish |
| 内部流程 | 接单后的报价、分配、跟踪、质检 | Quote、Assign、Track、Review |
| 售后服务 | 交付后的修改处理和客户回访 | Revise、Feedback |

**核心机制：** 每个 Skill 页面都是一个结构化表单。填写完成后点击"生成需求摘要"，系统会将所有信息整理成文本格式，可一键复制用于内部流转（发给写手、发到工作群、存档等）。

---

## 如何访问

1. 打开网站后，在浏览器地址栏访问 `你的域名/skills/` 即可进入 Skill 控制台
2. 或者在本地直接打开 `skills/index.html`
3. 控制台首页展示所有 Skill 卡片，点击即可进入对应工具

## 客户接入 Skill

> 这一层的 Skill 在与客户沟通时使用。当客户通过小红书/微信找到你，边聊边填写对应的采集表，确保不遗漏关键信息。

### 课程论文 Essay

**文件：** `skills/skill-essay.html`

**使用场景：** 客户需要代写课程论文、期中期末论文、读书报告等。

**采集字段：**

| 区块 | 字段 | 是否必填 | 说明 |
|------|------|---------|------|
| 客户信息 | 客户称呼 | 必填 | 方便后续沟通称呼 |
| | 联系方式 | 必填 | 微信号或手机号 |
| | 来源渠道 | 选填 | 小红书/微信/老客户推荐/其他 |
| 论文基本信息 | 学科/专业 | 必填 | 如：市场营销、教育学 |
| | 课程名称 | 选填 | 如：消费者行为学 |
| | 论文题目/方向 | 必填 | 已定题填题目，未定题填方向 |
| | 语言 | 必填 | 中文/英文/中英双语 |
| | 字数要求 | 必填 | 如：3000字 / 2000words |
| 格式与规范 | 引用格式 | 选填 | APA/MLA/Chicago/Harvard/GB/T 7714 |
| | 参考文献数量 | 选填 | 如：不少于10篇 |
| | 查重系统 | 选填 | 知网/Turnitin/维普/万方 |
| | 查重率要求 | 选填 | 如：低于15% |
| | 是否有格式模板 | 选填 | 有/没有 |
| 时间与预算 | 截止日期 | 必填 | 日期选择器 |
| | 是否加急 | 选填 | 正常/轻度加急/紧急/特急 |
| | 客户预算范围 | 选填 | 如：500左右 |
| 其他 | 特殊要求/补充说明 | 选填 | 老师偏好、理论框架等 |

**操作步骤：**
1. 与客户沟通时，打开此页面
2. 逐项填写（带 * 号为必填）
3. 点击「生成需求摘要」
4. 点击「复制到剪贴板」
5. 粘贴到工作群/写手对接/存档文件

**输出示例：**
```
【课程论文需求单】
============================
客户：张同学
联系方式：wxid_abc123
来源：小红书
---
学科/专业：市场营销
课程名称：消费者行为学
论文题目/方向：社交媒体对Z世代消费决策的影响研究
语言：中文
字数：5000字
---
引用格式：APA
参考文献数量：不少于15篇
查重系统：知网
查重率要求：低于20%
格式模板：有，会发送
---
截止日期：2026-03-25
是否加急：否（正常排期）
客户预算：600左右
---
备注：老师偏好用实证数据，需要有问卷调查部分
============================
采集时间：2026/3/10 14:30:00
```

## 内部流程 Skill

> 这一层的 Skill 在接单后由团队内部使用，客户不可见。

### 报价计算 Quote

**文件：** `skills/skill-quote.html`

**使用场景：** 根据需求采集的信息，计算并生成报价单。

**操作流程：**
1. 选择订单类型（课程论文/毕业论文/数据分析/留学文书/编程代做/润色降重）
2. 填写字数/工作量
3. 评估学科难度（普通/中等/较难/极难）
4. 选择语言和截止时间
5. 勾选附加服务（加急/查重报告/答辩PPT/代码讲解等）
6. 填入基础单价
7. 生成报价摘要

**输出内容：** 包含基础费用、难度系数、时间系数、附加费用、最终报价。

**定价参考：**
- 难度系数：普通 x1.0 / 中等 x1.2 / 较难 x1.5 / 极难 x2.0
- 时间系数：7天+ x1.0 / 3-7天 x1.2 / 1-3天 x1.5 / 24h内 x2.0

### 写手匹配 Assign

**文件：** `skills/skill-assign.html`

**使用场景：** 确认报价后，根据订单要求匹配合适的写手。

**操作流程：**
1. 录入订单编号和基本信息
2. 设置匹配条件：学历要求、专业匹配度、语言能力
3. 填写匹配到的写手信息：编号、评级、历史完成量
4. 生成分配记录

**匹配原则：**
- 学科精确匹配优先
- 高评级写手优先分配高难度/高价值订单
- 考虑写手当前工作量，避免过载

### 进度跟踪 Track

**文件：** `skills/skill-track.html`

**使用场景：** 跟踪订单从接单到完结的全过程。

**状态流转：**
```
待分配 -> 写作中 -> 初稿完成 -> 质检中 -> 已交付 -> 修改中(如需) -> 已完结
```

**操作流程：**
1. 输入订单编号
2. 更新当前状态和完成百分比
3. 记录里程碑（如"开题报告已完成"、"初稿已交付客户"）
4. 记录时间节点
5. 生成跟踪记录

**使用建议：**
- 每次状态变更时更新一次
- 重要节点务必记录时间
- 如有延期风险，提前在备注中标注

### 质检审核 Review

**文件：** `skills/skill-review.html`

**使用场景：** 写手交稿后、发给客户前的质量检查。

**质检清单（逐项检查）：**
- [ ] 查重率是否达标
- [ ] 格式是否规范（字体、行距、页边距）
- [ ] 引用是否正确（格式、数量）
- [ ] 内容是否切题（是否跑题、是否满足要求）
- [ ] 字数是否满足
- [ ] 语言是否流畅（语法、表达）
- [ ] 图表是否规范（如有）
- [ ] 参考文献格式是否统一

**操作流程：**
1. 逐项勾选质检清单
2. 填写查重率实际值
3. 给出质检评分（优秀/良好/合格/需修改）
4. 如有问题，在"问题描述"中详细说明
5. 选择处理意见（直接交付/小修后交付/需要返工/退回重写）
6. 生成质检报告

## 售后服务 Skill

> 这一层的 Skill 在交付后使用，处理修改请求和客户回访。

### 修改处理 Revise

**文件：** `skills/skill-revise.html`

**使用场景：** 客户收到稿件后提出修改意见。

**操作流程：**
1. 录入订单编号和客户信息
2. 选择修改轮次（第1轮/第2轮/第3轮/第4轮+）
3. 勾选修改类型（内容修改/格式调整/降重/补充内容/结构调整/其他）
4. 详细记录客户修改意见（原文粘贴）
5. 选择处理方式（原写手/换写手/主管处理）
6. 设置优先级和预计完成时间
7. 生成修改处理单

**注意事项：**
- 第 3 轮及以上修改需主管审核
- 如修改意见超出原始需求范围，需与客户沟通是否加价
- 所有修改意见建议保留原文，不要二次转述

### 客户回访 Feedback

**文件：** `skills/skill-feedback.html`

**使用场景：** 订单完结后进行客户满意度调查和复购引导。

**操作流程：**
1. 填写客户信息和订单编号
2. 记录满意度评分（整体/质量/沟通/时效，各 1-5 分）
3. 记录客户反馈原文
4. 评估后续跟进指标：好评意愿、复购意向、推荐意向
5. 生成回访记录

**回访时机建议：**
- 交付后 2-3 天（确认使用无问题）
- 成绩/结果出来后（拿到成绩/收到offer）
- 新学期开始前（复购提醒）

**好评引导话术参考：**
- "同学您好，请问上次的论文使用还顺利吗？"
- "如果满意的话，方便给个好评吗？后续订单可以优惠哦"

## 完整工作流程示例

以一个课程论文订单为例，展示 Skill 系统的完整使用流程：

```
第1步：客户通过小红书私信咨询
       -> 打开「课程论文 Essay」Skill
       -> 边聊边填写采集表
       -> 生成需求摘要，复制存档

第2步：评估需求，计算价格
       -> 打开「报价计算 Quote」Skill
       -> 填入需求信息，生成报价单
       -> 发给客户确认

第3步：客户确认，支付定金
       -> 打开「写手匹配 Assign」Skill
       -> 根据学科和难度匹配写手
       -> 生成分配记录，通知写手

第4步：写作过程中
       -> 打开「进度跟踪 Track」Skill
       -> 定期更新进度状态
       -> 如客户询问进度，可直接查看

第5步：写手交稿
       -> 打开「质检审核 Review」Skill
       -> 逐项检查质量
       -> 通过则交付客户

第6步：客户收到稿件
       -> 如需修改：打开「修改处理 Revise」Skill
       -> 记录修改意见，安排修改
       -> 修改完成后再次质检

第7步：订单完结
       -> 打开「客户回访 Feedback」Skill
       -> 记录满意度，引导好评
       -> 评估复购潜力
```

## 常见问题

**Q: Skill 页面的数据会保存吗？**
A: 当前版本数据不会自动保存。生成摘要后请务必复制保存。刷新页面或关闭浏览器后数据会丢失。

**Q: 可以在手机上使用吗？**
A: 可以。所有 Skill 页面都做了响应式适配，手机浏览器打开即可使用。

**Q: 如何添加新的 Skill？**
A: 复制一个现有的 Skill 页面（如 skill-essay.html），修改表单字段和生成摘要的逻辑，然后在 skills/index.html 中添加卡片链接即可。

**Q: 生成的摘要格式可以自定义吗？**
A: 可以。每个 Skill 页面底部的 `<script>` 中有 `generateSummary()` 函数，修改模板字符串即可调整输出格式。

**Q: Skill 系统和官网是什么关系？**
A: 官网（index.html）是面向客户的展示页面，Skill 系统（skills/ 目录）是内部运营工具。两者独立但部署在同一个仓库中。官网有一个"返回官网"的链接，Skill 控制台也有返回官网的按钮。
